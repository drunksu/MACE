Project Name: MACE-RL (Microstructure-Aware Conservative Execution via Reinforcement Learning)

Purpose:
This project aims to develop a microstructure-aware conservative offline reinforcement learning framework for optimal trade execution. The system integrates three key components:

1. A state-dependent action manifold learned via conditional normalizing flows to constrain feasible execution actions based on limit-order-book (LOB) microstructure.
2. An uncertainty-aware conservative value estimator using ensemble Q-functions to penalize out-of-distribution actions and mitigate offline extrapolation error.
3. A residual execution module that adapts dynamically to liquidity shocks and volatility regime shifts while respecting conservative manifold constraints.

The final agent will be trained using VerL (OpenAI’s reinforcement learning framework) and evaluated against classical execution baselines (TWAP, VWAP, Almgren–Chriss) and modern offline RL baselines (CQL, IQL, TD3+BC).

Dataset:
The primary dataset is the FI-2010 LOB dataset:
Ntakaris, Adamantios, et al. “Benchmark Dataset for Mid-Price Forecasting of Limit Order Book Data with Machine Learning Methods.”
This dataset contains high-frequency LOB snapshots suitable for microstructure feature extraction.

High-Level Requirements:
- Preprocess raw LOB data and generate microstructure-aware state representations (spread, depth, imbalance, volatility, queue metrics).
- Implement an offline execution simulation environment using historical LOB states.
- Train a conditional normalizing flow to model the state-conditioned feasible action manifold.
- Construct ensemble Q-networks and an uncertainty-aware conservative value estimator.
- Build a manifold-constrained execution policy with a residual adaptation module.
- Integrate all components into a VerL training pipeline using PPO/GRPO algorithms.
- Evaluate execution cost, slippage, and impact relative to baseline algorithms.

Deliverables:
- Full project codebase following the specified directory structure.
- Scripts for preprocessing, training, evaluating, and plotting.
- Reproducible experiment configurations using YAML files.
- Documentation for environment interfaces and the RL training loop.
